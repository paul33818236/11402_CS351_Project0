# 已知問題（Known Issues）

本文件列出目前專案在實作與驗證階段確認的問題、假設與限制，便於風險管理與後續修正。

## 1. 需求與範圍
- 目前需求是以學期作業規格為主，未納入產品級 API 穩定性與向後相容性保證。
- 需求仍可能被助教或指導老師追加邊界條件，需額外時間調整實作與測試。

## 2. 功能與實作
- `Twosum.c` 目前採簡單暴力或基本 hash map（視作業要求），尚未優化到最小時間複雜度 `O(n)`.
- 對於極端輸入（如超大陣列或極端整數範圍）尚未做壓力測試；可能存在整數溢出或效能瓶頸。
- 未明確定義輸入錯誤行為：如空陣列、NULL 指標或無解狀況，程式行為需與需求一致。

## 3. 測試相關
- 測試範圍目前以常見情境與單一正常案例為主，還缺少完整的邊界條件測試（負數、重複元素、全零、超長陣列）。
- 對於 C 語言記憶體管理（若有動態配置）尚未進行記憶體檢查工具（Valgrind、AddressSanitizer）驗證。
- 單元測試總體通過不等於驗收測試 100% 全覆蓋，還需補足 `acceptance_tests.md` 所列情境。

## 4. CI / 平台
- 目前未整合 CI（GitHub Actions / GitLab CI），無法自動化執行每次 PR 測試。
- 平台相依性：本地開發使用 Ubuntu，其他測試環境可能需要調整 Makefile/編譯指令。

## 5. 文件與專案管理
- 文件尚需持續對齊規格：SRS、SDS 和 traceability 更新頻率須保持一致。
- 目前未執行正式 code review，風險包括漏洞、命名不一致、缺少注釋或錯誤處理。

## 6. 後續改進建議
- 補齊 `test_plan.md` 的測試矩陣與驗收標準
- 加入效能分析：時間、記憶體、邊界效能測試結果
- 導入 CI 管道與覆蓋率報告，並標記修復日期
- 處理上述已知問題並在 `known_issues.md` 註明狀態（open / fixed / postponed）
