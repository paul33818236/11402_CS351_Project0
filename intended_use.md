# Intended Use（預期用途）

本文件說明本專案的預期使用場景、適用對象與不適用情境，以便評分、驗收與後續延伸。

## 1. 預期使用場景
- 請助教和評分者執行兩數求和（Two Sum）功能驗收測試。
- 作為作業進度與開發文件驗收依據，包含需求、設計、測試與部署。
- 展示 C 語言基本演算法實作、測試設計、文件完整性與專案管理能力。

## 2. 目標使用者
- CS351 課程學生（開發者）
- 助教/講師（評分者）
- 專案合作成員（跨組協作）

## 3. 核心功能
- `Twosum.c`：讀取整數數列與目標值，輸出任意一對符合條件的索引。
- 單元測試與驗收測試文件：驗證基本用例與邊界條件。
- 文件資源：`SRS.md`, `SDS.md`, `plan.md`, `test_plan.md`, `traceability.md`, `acceptance_tests.md`, `known_issues.md`, `deploy.md`。

## 4. 不適用範圍
- 非課程作業：不適合作為生產級 API 或大規模服務使用。
- 顯式安全強化（未做防攻擊、沙盒隔離、用戶認證）。
- 高效能大資料環境（如 million+ 輸入、要求實時低延遲等）。

## 5. 使用流程建議
1. 先閱讀 `README.md`、`plan.md` 了解專案架構。
2. 執行 `make test` 或手動測試以驗證 `Twosum` 正確性。
3. 檢視 `acceptance_tests.md` 與 `known_issues.md`，確認是否符合評分標準。
4. 若需調整輸入行為（無解/錯誤），同步更新 `SRS.md`、測試與程式。

## 6. 版本與延伸建議
- 目前版本：v1.0（作業驗收版）
- 延伸項目：加入 CLI/GUI 參數解析、記憶體可以動態釋放、快速 hash 方案、CI/CD 等。
