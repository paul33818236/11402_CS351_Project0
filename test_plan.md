# Test Plan (測試計劃)

本文件定義 `Twosum` 專案的測試策略、測試案例與驗證方式，配合 `SRS`、`SDS` 與 `acceptance_tests`。

## 1. 測試目標
- 驗證 Two Sum 功能正確性
- 驗證邊界情境與例外處理
- 確保測試流程可再現且可自動化

## 2. 測試範圍
- 單元測試：函式 `twosum` 的每個條件
- 集成測試：程式輸入/輸出處理與錯誤訊息
- 驗收測試：`acceptance_tests.md` A1~A10
- 迴歸測試：先前 bug case

## 3. 測試層級
1. 單元測試
   - `twosum` 函式輸出正確索引（或 NULL）
   - 非法輸入處理
2. 集成測試
   - CLI / 程式流程從輸入到結果輸出
3. 驗收測試
   - 依 `acceptance_tests.md` 評估

## 4. 優先測試案例
| ID | 類別 | 輸入 | 預期輸出 | 通過條件 |

| T1 | 基本 | [2,7,11,15],9 | [0,1] | 兩數和 9 |

## 5. 測試實作方式
- 使用 gtest / cmocka / Unity (若無框架可編寫自定義 assert)
- 支援 `make test` 執行（標準腳本）：
  - `gcc -std=c11 -Wall -Wextra Twosum.c test_twosum.c -o test_twosum`
  - `./test_twosum`
- 測試程式須列印測試結果總表（pass/fail）

## 6. 測試記錄與報告
- 每次測試執行輸出 log 推至 `test_report.txt`
- 包含測試日期、案例、狀態、錯誤訊息、修復狀態

## 7. 測試風險與緩解
- API 定義變更：先更新 `SRS` 並補測試
- 邊界條件遺漏：增加驗收測試/例外 case
- CI 異常：設置本地及遠端雙路測試

## 8. 追蹤與驗收
- 在 `traceability.md` 撰寫測試->需求映射
- `known_issues.md` 記錄失敗案例與解決狀態
- 最終應由評分者確認所有必須通過案例達標
