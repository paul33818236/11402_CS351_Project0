# Software Requirements Specification (SRS)

## 1. 引言
本文件為 `CS351 專案 0` 的需求規格，涵蓋 Two Sum 演算法核心功能、輸入輸出行為、非功能需求以及驗證標準。

### 1.1 目的
提供清晰可驗證的需求說明，保障開發過程與評分標準一致，並作為測試與設計的基礎。

### 1.2 範圍
- 功能：Two Sum, 輸入整數陣列 `nums` 與目標值 `target`，回傳任意一組索引 `i < j` 使 `nums[i] + nums[j] == target`。
- 輸出：有效配對索引或無解/錯誤狀態。

## 2. 功能需求
### 2.1 基本功能
1. 使用者可輸入整數陣列與目標值。
2. 系統返回配對索引，或沒有符合配對時返回無解。
3. 若輸入資料不合法（空陣列、NULL、長度不足2等），系統需回傳錯誤或明確無解。

### 2.2 邊界條件與例外
- 重複元素：可使用同樣數值但不同索引，例如 `[3,3], target=6`。
- 包含負數 (
3、0等)：應支援。
- 空陣列、單元素陣列：回傳無解。
- 非整數或指標錯誤：可定義異常輸出。
- 可能溢出：若使用 `int` 應以 `long long` 暫存計算避免溢位判斷。

## 3. 非功能需求
- 可移植：純 C99 / C11 標準。
- 可靠性：所有必須使用案例 100% 通過。
- 性能：目標 `O(n)` (Hash map)；至少 `O(n^2)` 可接受。
- 可測試性：必備單元測試、驗收測試、測試報告。

## 4. 外部介面需求
### 4.1 使用者介面
- CLI or 測試函式皆可，需提供範例命令與輸出結果說明。

### 4.2 程式介面
- `int* twosum(int* nums, int numsSize, int target, int* returnSize);`
- 若需額外夾帶輸出：`print_result` 或 `print_error`。

## 5. 系統行為
- 讀入輸入資料 -> 演算法計算 -> 比對結果 -> 輸出索引或訊息。
- 如有多組解，可回傳任意一組。
- 回傳結果需為 `returnSize==2` 否則 `0` 表示無解。

## 6. 測試需求
- 單元測試：A1~A10 驗收測試場景。
- 邊界測試：空陣列、大小1、重複、極大極小值。
- 維護驗收：`acceptance_tests.md` 文件展示。

## 7. 依賴與假設
- 假設使用者提供的 `nums` 為可存取有效記憶體空間。
- 假設本作業不需安全沙盒或多工同時需求。

## 8. 可追溯性
- 需求映射至 `test_plan.md`、`SDS.md` 與 `acceptance_tests.md`。
- 發現問題紀錄於 `known_issues.md`，更新狀態追蹤。
