# Acceptance Tests (驗收測試)

本文件定義針對 `Twosum.c`（Two Sum）功能之驗收測試案例與通過標準。

## 1. 系統行為描述
- 目標：輸入整數陣列與目標值，回傳任意一組索引 i, j（i < j），使得 nums[i] + nums[j] == target。
- 輸出：兩數索引（0 base）或指定錯誤狀態（無解/輸入錯誤）。
- 假設：陣列元素可包含正數、負數、零；索引應為整數且唯一。

## 2. 驗收測試矩陣
| 測試編號 | 測試描述 | 輸入 | 預期結果 | 是否須通過 |
|---|---|---|---|---|
| A1 | 標準案例 | [2,7,11,15], target=9 | [0,1]（或等價） | 必須通過 |
| A2 | 重複元素 | [3,3], target=6 | [0,1] | 必須通過 |
| A3 | 含負數 | [-1,2,3,-4], target=-5 | [0,3] | 必須通過 |
| A4 | 多種候選解 | [1,2,3,4], target=5 | [0,3] 或 [1,2] | 必須通過 |
| A5 | 無解情境 | [1,2,3], target=7 | 明確回傳無解 / 空結果 | 必須通過 |
| A6 | 空陣列 | [], target=0 | 無解或錯誤（視需求） | 建議通過 |
| A7 | 單元素陣列 | [5], target=5 | 無解或錯誤 | 建議通過 |
| A8 | 大值檢測 | [1000000, 500000, 500000], target=1500000 | [1,2] | 建議通過 |
| A9 | 全零 | [0,0,0,0], target=0 | 任一合法配對（如 [0,1]） | 建議通過 |
| A10 |  overflow 檢查 | [2147483647, -1, -2147483648], target=-1 | [0,1] 或其他安全實作 (若有) | 建議通過 |

## 3. 驗收標準
- 所有“必須通過”的案例需通過。
- “建議通過”案例應至少 80% 成功；若理出邊界行為則更佳。
- 輸出格式需明確且可重複執行（CLI 或單元測試套件）。

## 4. 測試執行與紀錄
- 使用 `make test` 或類似指令執行測試。
- 測試資料與結果應加到 `test_plan.md`/`test_report` 中。
- 異常案例需說明是否屬於需求內行為（無解、輸入錯誤、溢位）。

## 5. 追蹤與改善
- 若某驗收測試失敗，記錄錯誤訊息與輸出，並將原因標記 `已修正/待修正`。
- 對回報無解、空陣列、溢出等行為，統一需求文件（SRS）、程式註解與 `known_issues.md`。
- 最終執行完整套驗收測試後，將 `acceptance_tests.md` 更新為「已通過」或「未通過（附註）」。
